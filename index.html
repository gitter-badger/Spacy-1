MinimalSpacy

Welcome to Spacy,

an abstraction layer for sharing and developing vector space algorithms. It is motivated by the following ideas:

Sharing

The main aim of this library is to facilitate sharing of algorithms. Wit Spacy researchers and users can develop and share algorithms without being tied to specific frameworks for the problem definition.

Cleaner Code

The separation of problem definition and solution algorithm helps in revealing the algorithmic structure in its implementation. Moreover it helps enforcing the separation of responsibilities.

Rapid Prototyping

Spacy uses type erasure for the definition of interfaces. Consequently it supports duck-typing and value-semantics. Thus it becomes easy to develop new algorithms.

Dependencies

Spacy depends on Boost.TypeErasure (available from Version 1.54.0).

In Ubuntu you can install boost with

sudo apt-get install libboost-dev


Installation

Spacy can be easily installed with cmake. Boost.TypeErasure and, if present, FEniCS (dolfin) should be detected automatically.

The usual installation procedure is:

  • cg Spacy_root_dir
  • mkdir build
  • cd build
  • cmake ..
  • make

Algorithms

Conjugate Gradients

There are four different conjugate gradient implementations all with exchangeable termination criterion. The default criterion is based on the relative energy error.

Available implementations:

  • Conjugate Gradients (CG)
  • Truncated Conjugate Gradients (TCG)
  • Regularized Conjugate Gradients (RCG)
  • Truncated Regularized Conjugate Gradients (TRCG)


Newton Methods

There is a generic Newton method that allows to customize termination criterion and damping strategy.

Available implementations:

  • a local Newton method
  • an affine covariant Newton method
  • an affine contravariant Newton method


Composite Step Methods

There is a composite step method for nonconvex optimization, where the normal solver can be customized.

Available Implementations:

  • an affine covariant composite method


Vector Spaces

  • a real space
  • a product space that admits separation of primal and dual variables (i.e. Lagrange multipliers).

Adapters
